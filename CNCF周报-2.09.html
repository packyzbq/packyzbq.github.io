CNCF周报-2.09

Posted on February 11, 2020

要闻

containerd 1.3.3 发布

这次 containerd 发布了第三个patch版本，包括了多个修复和更新：

  • runtime v2中，将runc shim 中关闭平台的操作移动至 Shutdown 中，确保平台只被关闭一次 #3907
  • 修复了 eventfd leak 的问题 #3961
  • 修复API筛选器，保证正确处理并返回解析错误 #3950

其他更新见 release note

CNCF 发布了 containerd 的旅程报告

链接：https://www.cncf.io/blog/2020/02/05/announcing-the-containerd-project-journey-report/

主要讲了 containerd 从开始到cncf毕业的旅程，以及迄今为止 containerd 中的代码贡献度、发展速度和项目文档发展等。有兴趣的小伙伴可以看一下

docker 发布 19.03.6-rc2 版本

  • containerd 更新至 1.2.12 版本
  • 修复了 usernamespace enabled 时目录权限的问题 #40440
  • 缩短了 exec-root 中的 controller ID，防止到达 UNIX_PATH_MAX

详见 release note

上游动态

在 Kubernetes v1.18 中，会将之前已标注过期的各类 generator 都移除掉。 也就是说，自 v1.18 起使用 kubectl run 命令主要就是创建 Pod 了，而不会创建多余的 deploy 之类的。

项目推荐

Knative

Knative 扩展了Kubernetes，以提供一组中间件组件，这些组件对于构建可在任何地方运行的现代，以源为中心和基于容器的应用程序必不可少：本地，云中甚至第三方数据中心。

项目地址：https://github.com/knative
