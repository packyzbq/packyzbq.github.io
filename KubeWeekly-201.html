CNCF 周报 2020-2-2

Posted on February 2, 2020

要闻

Kubernetes 发布 v1.18.0-alpha.2 版本

这次更新变更东西比较多，以下是我个人认为比较重要的点：

  • Secret 和 ConfigMap 引入 alpha 属性 Immutable，需要开启ImmutableEphemeralVolumes 的 Feature Gate
  • kubectl apply -f <file> --prune -n <namespace> 此指令增加接收特定的 namespace。 #85613
  • kubelet 将一些 metrics endpoint 设为过期
  • 现在可以将 --node-ip :: 传递给kubelet，默认设置 IPv6 地址为主地址。#85850
  • kubeadm 在拉取镜像失败后，可以增加重试次数，默认为5次
  • kubeadm 使用bind-address选项配置kube-controller-manager和kube-scheduler 的http探针
  • 调度策略部分抛弃了 AlwaysCheckAllPredicates API
  • hollow node 不再使用远程的 CRI，使用 fake CRI #86425 #85879

change log

Linkerd 发布 edge-20.1.4 版本

此版本针对 cli 增加了三处更新

release note

CoreDNS v1.6.7 发布

本周 CoreDNS 有小版本更新，主要更新如下：

  • plugin/{kubernetes, etcd}: 允许通过 CNAME 解析 TXT 记录。这个更新是为 backend_lookup.go 中的 TXT 方法增加一个参数。目前 kubernetes 和 etcd 插件的相关调用已经更新，如果有第三方或自己实现的插件，需要注意。

之前在 twitter 上发起了一个调查，是本地 K8s 开发者最常用的工具，分别是

目前最多使用的是 Minikube，Kind 之前也刚刚发布了新版本 v0.7.0，日后可以出一篇blog去专门对比一下这三个工具

Kubernetes 在上周相继发布了 v1.15.8，v1.16.5 以及 v1.17.1 三个版本，但这几个版本的构建信息有误，导致这周又相继发布了 v1.15.9, v1.16.6 以及 v1.17.2 版本。

项目推荐

Chaos Mesh

在分布式计算的世界中，群集随时随地都可能发生故障。传统上，我们使用单元测试和集成测试来确保系统已准备就绪，但是随着集群规模，复杂性数量和数据量（基于PB级别）的增加，这些测试仅涵盖冰山一角。为了更好地识别系统漏洞并提高弹性，Netflix发明了Chaos Monkey，并将各种类型的故障注入基础架构和业务系统。这就是Chaos Engineering的起源。

Chaos Mesh是一个通用的Chaos Engineering（混沌工程）平台，具有针对Kubernetes上复杂系统的全方位故障注入方法，涵盖Pod，网络，文件系统甚至内核中的故障。由 TiDB 团队开发。

github: https://github.com/pingcap/chaos-mesh
文章链接: https://pingcap.com/blog/chaos-mesh-your-chaos-engineering-solution-for-system-resiliency-on-kubernetes/

Vitness

Vitess是一个用于部署，扩展和管理大型开源数据库实例集群的数据库解决方案。它目前支持MySQL和MariaDB。它在架构上可以像在专用硬件上一样有效地在公共或私有云架构中运行。它结合并扩展了许多重要的SQL功能以及NoSQL数据库的可伸缩性。

github: https://github.com/vitessio/vitess

KubeInvaders

在 K8s 官网中，近日发布了一个blog，介绍了一个混沌工程工具，Kubenvaders是一个针对Kubernetes和Openshift的游戏化混沌工程工具，可通过一种有趣的方式帮助测试Kubernetes集群的弹性。

项目地址在 https://github.com/lucky-sideburn/KubeInvaders， 有兴趣的可以搞搞看

log-pilot

此工具用于收集容器中的标准日志以及指定路径的日志。大体也还是使用 fluntd 或 filebeat 进行收集，对于容器中指定路径的日志文件，大体思路是使用 emptyDir 将日志文件路径暴露出来，使其可在宿主机上看到，再进行追踪。

项目地址 https://github.com/AliyunContainerService/log-pilot
