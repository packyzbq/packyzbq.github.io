kubelet源码分析--启动篇

Posted on February 5, 2020

在Kubernetes集群中，每个Node节点（又称Minion）上都会启动一个Kubelet服务进行。该进程用于处理Master节点下发到本节点的任务，管理Pod及Pod中的容器。每个Kubelet进程会在API Server上注册节点自身信息，定期向Master节点汇报节点资源的使用情况，并通过cAdvise监控容器和节点资源。

调用的 func run 将会执行以下一系列操作：

  5. 初始化 container manager
  6. 进行权限验证，需要以 uid 为 0 的用户启动。
  7. 调用 RunKubelet

调用 RunKubelet

RunKubelet 主要流程：

  1. 获取主机名
  2. 建立并初始化 event recorder
  3. 获取以下资源(均读取自 KubeletFlags)：
    • hostNetworkSources
    • hostPIDSources
    • hostIPCSources
    • privilegedSources
  4. 若设置 runonce 参数，则只拉取一次容器组配置，并在启动容器组后退出，否则将以 server 形式保持

  • 对于runonce，首先创建所需的目录，监听 pod update 信息，得到 pod 信息后，创建pod 并返回他们的状态

  • 以 server 模式启动，流程如下：

    1. 检查 logserver 以及 apiserver 是否可用
    2. 如有 cloud provider 配置， 则启动cloudResourceSyncManager
    3. 启动 volumeManager
    4. 调用 kubelet.syncNodeStatus同步 node 状态
    5. 调用 kubelet.updateRuntimeUp同步 runtimeState 中的容器的 runtime uptime
    6. 开启循环，同步 iptables 规则
    7. 启动一个用于“杀死pod” 的 goroutine
    8. 启动 statusManager 和 probeManager （都是无限循环的同步机制）
    9. 启动 runtimeclass manager （注意这里与替换底层容器相关）
    10. 启动 pleg （pod lifecycle event generator）

    至此，kubelet 整体的启动流程完毕，进入无限循环中，实时同步不同组件的状态。同时也对端口进行监听，响应 http 请求。
